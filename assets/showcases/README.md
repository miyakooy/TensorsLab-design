# Showcases

This folder should contain reusable visual examples for TensorsLab Design.

## Purpose

Showcases help the skill do three things:

1. show the user possible directions quickly
2. give the model a visual grammar to follow
3. prevent bland, average-looking output

## Planned showcase groups

- cover / social hero
- pitch deck slides
- infographic layouts
- website or landing-page sections
- product launch visuals
- comparison / variant boards

## Manual migration note

Any showcase copied from the source project must be reviewed before reuse. If it is too tied to the source project’s identity, rebuild it for TensorsLab instead.
